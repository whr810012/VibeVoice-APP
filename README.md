# VibeVoice Desktop 系统说明文档

## 1. 项目定位
**VibeVoice Desktop** 是基于 **Electron** 和 **Vue 3** 框架对原始 [VibeVoice](https://github.com/vibe-voice/VibeVoice) 语音系统进行的深度桌面化封装。本项目旨在将强大的 AI 语音识别（ASR）与语音合成（TTS）能力带给普通用户，提供一个稳定、直观、且无需复杂配置的生产力工具。

## 2. 核心架构
系统采用“前后端分离 + 桌面容器”的架构：
- **前端 (Renderer Process)**: 使用 Vue 3 + TypeScript + Element Plus 构建。负责所有 UI 交互、实时状态显示（如 ASR 进度条）、历史记录预览及播放。
- **主进程 (Main Process)**: Electron 核心。负责嵌入式 Python 运行时管理、后端 FastAPI 进程的生命周期监控（启动、停止、异常重启）以及文件系统导出操作。
- **后端 (Python Backend)**: 基于 FastAPI 封装的 VibeVoice 核心模型服务。通过多线程 Job Queue 处理长音频 ASR 任务，并提供流式 TTS 接口。

## 3. 核心优势（对比原始 VibeVoice）

### 🚀 一键部署，零环境依赖
- **现状**: 原始版本需要用户手动安装 Python、Git，处理 CUDA 驱动，并解决 pip 依赖冲突。
- **优势**: 本系统集成了 **嵌入式 Python 环境**。用户只需解压或安装 `.exe` 包，双击即可运行，无需预装任何编程环境。

### 📊 全流程可视化交互
- **现状**: 原始版本主要通过命令行或基础 Web 界面操作，反馈较为单一。
- **优势**: 
    - **ASR 进度条**: 实时显示音频识别进度，支持随时取消任务。
    - **TTS 历史记录**: 自动保存所有合成记录，支持在线预览播放、下载及重命名。
    - **配置面板**: 图形化修改端口、运行设备（CPU/GPU），即时生效。

### 🛡️ 智能后端进程管理
- **现状**: 原始版本需要手动启动后端 Python 脚本，且修改配置后需手动重启。
- **优势**: Electron 自动托管 Python 后端。当用户在设置页切换“计算设备”时，Electron 会**自动重启后端进程**并应用新环境，确保模型加载无缝切换。

### 📂 增强的文件管理与导出
- **现状**: 原始生成的文件分散在临时文件夹中，查找困难。
- **优势**: 
    - 内置“历史记录”管理，按时间排序。
    - **一键批量导出**: 支持用户选择自定义目录，一键将所有 TTS 音频或 ASR 结果导出到本地指定位置。

### ⚙️ 硬件自适应
- 支持动态检测 CUDA 环境。
- 允许用户在 UI 上自由切换 **CUDA 加速** 与 **CPU 运行** 模式，最大化利用硬件性能，同时保证低配电脑的可用性。

## 4. 总结
本项目不仅是 VibeVoice 的一个“壳”，而是一个**完整的桌面端解决方案**。它通过 Electron 弥补了 Python 工具在用户分发和交互体验上的短板，使 VibeVoice 能够从“实验室工具”进化为“大众消费级应用”。
