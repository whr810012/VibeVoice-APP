# VibeVoice Desktop 0.1.1 Release Notes

发布日期：2026-04-16  
安装包：`vibevoice-app/release-0.1.1/VibeVoice Desktop Setup 0.1.1.exe`

## 本次更新重点

### 1) UI 视觉系统升级
- 建立统一设计变量体系（颜色、间距、圆角、阴影）。
- 页面结构统一为“导航 + 工作区 + 状态反馈”模式，信息层级更清晰。
- 卡片、按钮、表格与表单视觉风格统一，整体更接近商业化桌面应用体验。

### 2) 主题系统闭环
- 新增主题模式：`跟随系统 / 浅色 / 深色`。
- 主题切换即时生效，并可持久化保存。
- 设置页支持“一键恢复默认外观（跟随系统）”。

### 3) 可访问性与可读性增强
- 增加 `focus-visible` 焦点样式，提升键盘操作可见性。
- 新增高对比模式与强制颜色模式适配。
- 暗色主题下语义色（主色/成功/警告/危险）与文本对比度进行专项校准。

### 4) 历史记录与交互体验优化
- 历史列表支持空状态引导，一键跳转创建任务。
- 操作按钮语义统一（播放/导出/删除），并补充语义图标。
- 结果区、加载态与提示文案风格统一，减少“状态割裂”。

### 5) 开发稳定性修复
- 修复开发模式偶发白屏：Electron 端增加加载重试与加载失败日志。
- 修复 Vite 误扫描外部目录问题：限制依赖入口并忽略 `release/python-win` 目录。
- 优化打包流程，降低 Windows 下构建产物文件锁冲突概率。

## 兼容性说明
- 推荐 Node.js：`v20 LTS`（最低 `v18+`）。
- 当前安装包仍使用默认 Electron 图标（尚未配置 `build/icon.ico`）。

## 已知事项
- `requests` 相关 `RequestsDependencyWarning` 来自本机 Python 环境依赖版本提示，不影响桌面端核心 UI 功能使用。
- Windows 环境中若出现旧产物锁文件，请关闭占用中的 Electron/Node 进程后重试打包。

## 升级建议
- 0.1.0 用户可直接安装 0.1.1 进行覆盖升级。
- 升级后建议验证：主题切换、ASR 页面、TTS 页面、历史导出、设置保存重启。
